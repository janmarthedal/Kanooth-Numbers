Sputsoft Numbers

Sputsoft Numbers is a C++ library for doing multiple-precision arithmetic. The underlying theory and design can be found among the multiple-precision articles at sputsoft.com.

The library is header-only and highly portable, relying only on some boost libraries. It is possible, however, to obtain highly optimized code by choosing the GMP backend. This requires no code changes, but only setting a preprocessor macro and linking to the GMP library (see below).

Installation and Basic Usage

Since the library is header-only, you only need to place all the files from the library somewhere locally and instruct your C++ compiler to use the directory containing the sputsoft folder as an include directory. The boost libraries needed are integer traits and shared pointer. So, assuming that boost is somewhere among your compiler's standard include search paths, you should be able to compile using, e.g., 

  g++ -I. test.cpp

if using the GNU C++ compiler and the current directory contains the sputsoft folder.

By default, the library uses generic and portable C++ code for the low-level arithmetic routines. If you wish to use the low-level routines from the GMP library, you can compile the program from above as 

  g++ -I. test.cpp -DSPUTSOFT_USE_GMP -lgmp

assuming that the GMP library is installed and available on the platform.

Unless explicitly stated, every data type and function from reside in the sputsoft::numbers namespace.

Data Types

The data types currently supported are natural_number and integer. They are initialized when declared and automatically clean up when going out of scope.

All functions operating on these numbers have the form name(output numbers, ..., input numbers, ...). For instance, add(z, x, y) adds x and y and puts the result into z. Most of these functions also have value-returning versions that create a temporary object to put the result into. For instance, the following two lines do the same thing 

  add(z, x, y)
  set(z, add(x, y))

The first of these statements, however, is likely to be more efficient since creating a temporary may be avoidable.

The operators =, +, -, *, /, %, ⟨⟨, and ⟩⟩ have been overloaded to represent set, add, sub, mul, div, rem, bit_shift_left, and bit_shift_right, respectively.

Natural Number

Natural numbers can be initialized in four ways: No initializer (set to zero), set to built-in unsigned integer, set to C/C++ string (const char* or std::string), or set to the value of another variable of the same type. For instance, 

  natural_number a;
  natural_number b = 2u;
  natural_number c = "1234";
  natural_number d = b;

Source Code and Downloading

The current release is version 0.2 and is hosted at SourceForge. The library can be downloaded or the latest source code can be browsed.

Bug reports, suggestions for improvements, and other comments are very welcome.
